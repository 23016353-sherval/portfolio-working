Character Catalouge

~~INTRODUCTION~~

This project was done after I learned some basics of Mobile App Development using REACT. It was coded on Webstorm and displayed on an emulator of the pixel 8. We were tasked with creating this simple app to display whatever we wanted, so I choose to display the different classes of my favourite game, Dragon Nest. The reason I chose Dragon Nest is because the wiki and official sites don't have like a clear list of the different classes so I wanted to make one for myself so I can decide which class I want to pick next time I play a new character. It's a very simple app but I want to maybe give it a design overhaul and add some features in to upgrade the app during my free time.

~~DESIGN~~

SIMPLISTIC DESIGN

The design is very simple. The "Add Character" button is self-explanatory. For editing information, I used Touchable Opacity so you can just click which item you want to edit instead of clicking a separate button, giving it a cleaner look. The character classes are color coded to easily tell which classes belong to which character, I chose those colours specifically for each class because most of these are the colours of their class symbols, I only gave different colours to those with like too similar colours to others (e,g, Kali & Lancea - I feel the colour fits the Lancea more than the Kali since it is the colour of her attacks). I might try and change the overall design of the app when I have time.

~~DEVELOPMENT~~

FUNCTIONS

There were 4 pieces of information to render, the name of the Class, what specialization it is, a short description of the class, and the image link. Each has its own input field as well as 1 more dropdown field that determines which section it will go. Since this project was done near the beginning, there are only the basic Create(C), Render(R), Update(U) & Delete(D) functions. For creating and rendering, I added the Tempest under the section "Archer Classes" and after submitting, it is added to the list below the silver hunter.

PROBLEMS

There were weird problems along the way, 1 of them was about the images, for some reason the normal image types like PNGs and JPGs would just stop working and not load, but it only doesn't work for me because for some reason, my friends all have no problem loading the images and everything even for my own project so I found that a bit annoying, but either way, since I needed the images to load for the video submission I found a work around for this problem, which was to use data:image photos, like those with the super long image links, instead of the normal PNGs and JPGs, those links are way too long but if it works, it stays. At least until my images work again.

Another Problem is that the whole emulator itself would like stop loading, and this only starts like 2 hours before the submission deadline. I almost missed the submission window like 3 times because of this. But I found a fix for this, I can give the emulator a cold boot from Andoid Studio. Once I rebooted it the app finally work again. I finally know what to do when the emulator starts dying on me again.
